Time Integration

HEGEL provides explicit, implicit, and operator-splitting time integrators. The integrator is selected via the NUM_TIME_INT keyword. Implicit methods additionally require NUM_LHS_JAC to specify the Jacobian approximation.

Explicit Methods

Forward Euler

First-order explicit method:

\[U^{n+1} = U^n + \Delta t \, R(U^n)\]

where \(R(U)\) is the spatial residual (RHS).

Input:

NUM_TIME_INT = FE

Source files: class_fe.F90, fe.F90

Classical Runge-Kutta (RK2, RK4)

Standard multi-stage Runge-Kutta methods using the Butcher tableau representation. RK2 is the midpoint method; RK4 is the classical 4-stage method.

RK4 Butcher tableau:

\[\begin{split}\begin{array}{c|cccc} 0 & & & & \\ 1/2 & 1/2 & & & \\ 1/2 & 0 & 1/2 & & \\ 1 & 0 & 0 & 1 & \\ \hline & 1/6 & 1/3 & 1/3 & 1/6 \end{array}\end{split}\]

Input:

NUM_TIME_INT = RK4   # or RK2
NUM_RUN_MODE = CONSTANT_CFL
NUM_CFL      = 0.5

Source files: class_rk.F90, rk.F90

SSP Runge-Kutta (TVD)

Strong Stability Preserving (SSP) variants that maintain the Total Variation Diminishing (TVD) property of the spatial discretization. Essential when using limiters near discontinuities.

SSP-RK3 (Shu-Osher form):

\[\begin{split}U^{(1)} &= U^n + \Delta t \, R(U^n) \\ U^{(2)} &= \tfrac{3}{4} U^n + \tfrac{1}{4} U^{(1)} + \tfrac{1}{4} \Delta t \, R(U^{(1)}) \\ U^{n+1} &= \tfrac{1}{3} U^n + \tfrac{2}{3} U^{(2)} + \tfrac{2}{3} \Delta t \, R(U^{(2)})\end{split}\]

Input:

NUM_TIME_INT = RK3_SSP   # or RK2_SSP, RK2_TVD, RK3_TVD

Source files: class_rk_ssp.F90, rk_ssp.F90

Note

RK2_TVD and RK2_SSP are identical; likewise RK3_TVD and RK3_SSP.

Low-Storage Runge-Kutta

Memory-efficient RK variants using the 2N-storage formulation. Only two solution-sized arrays are needed regardless of the number of stages.

Input:

NUM_TIME_INT = RK_LS

Source files: class_rk_ls.F90, rk_ls.F90

Low-Storage RK (Point-Implicit)

Low-storage RK with point-implicit treatment of the source term diagonal. Improves stability for stiff source terms without the cost of a full implicit solve.

Input:

NUM_TIME_INT = RK_LS_PI

Source files: class_rk_ls_pi.F90, rk_ls_pi.F90

Warning

Not available when sponge zones are present.

Strang Splitting

Second-order operator splitting that separates the convective (flux) operator from the stiff source (chemistry/relaxation) operator:

\[U^{n+1} = S_{\Delta t/2} \circ F_{\Delta t} \circ S_{\Delta t/2} \, U^n\]

The source operator \(S\) uses an external ODE integrator (CVODE/LSODE via SUNDIALS) to handle chemical stiffness. Default ODE tolerances: rtol = 1e-6, atol = 1e-9.

Input:

NUM_TIME_INT = STRANG   # or OS
NUM_RUN_MODE = CONSTANT_DT
NUM_DT       = 1.0e-8

Source files: class_Strang.F90, Strang.F90, Strang_flux.F90, Strang_source.F90

Note

Requires constant time-step mode (CONSTANT_DT). Requires SUNDIALS at compile time (--with-sundials).

Implicit Methods

Implicit methods solve a linear system at each time step. The structure of the left-hand side (LHS) matrix is controlled by NUM_LHS_JAC:

Keyword

Approximation

Description

FULL

Full sparse Jacobian

Solved by PETSc GMRES. Most accurate but most expensive.

DIAG

Block-diagonal (point-implicit)

Each cell inverted independently. Fast, low memory.

DPLR

Data-Parallel Line Relaxation

Tridiagonal block solves along grid lines. Good balance of accuracy/cost.

Backward Euler

First-order implicit:

\[\frac{U^{n+1} - U^n}{\Delta t} = R(U^{n+1})\]

Linearised as:

\[\left[\frac{V}{\Delta t} I - \frac{\partial R}{\partial U}\right] \Delta U = R(U^n)\]

Three LHS variants are available:

Backward Euler + FULL:

NUM_TIME_INT = BE
NUM_LHS_JAC  = FULL
NUM_LSS_ALG  = GMRES
NUM_LSS_PC   = ASM
NUM_LSS_RTOL = 1.d-12
NUM_CFL      = 50

Backward Euler + DIAG (point-implicit):

NUM_TIME_INT = BE
NUM_LHS_JAC  = DIAG
NUM_CFL      = 10

Backward Euler + DPLR (line relaxation):

NUM_TIME_INT = BE
NUM_LHS_JAC  = DPLR
NUM_CFL      = 100

Source files:

  • class_be.F90, be.F90 (FULL)

  • class_be_diag.F90, be_diag.F90 (DIAG)

  • class_be_dplr.F90, be_dplr.F90 (DPLR)

Note

DPLR is not available for parallel 1D simulations. Sponge zones are not supported with BE.

Dual-Time Stepping

For time-accurate implicit simulations, HEGEL uses dual-time stepping: an inner pseudo-time iteration drives the solution to convergence at each physical time level. Enable with:

NUM_DUAL_TIME = T

Key parameters:

Keyword

Default

Description

NUM_DUAL_TIME_MAX_IT

–

Maximum sub-iterations per physical time step

NUM_DUAL_TIME_RES

–

Convergence threshold (residual L2 norm)

NUM_DUAL_TIME_WRMS_LIM

–

Convergence threshold (WRMS error)

NUM_DUAL_TIME_WRMS_RTOL

–

Relative tolerance for WRMS test

NUM_DUAL_TIME_WRMS_ATOL

–

Absolute tolerance for WRMS test

Three-Point Backward (BDF2)

Second-order implicit time-accurate method using three time levels:

\[\frac{3 U^{n+1} - 4 U^n + U^{n-1}}{2 \Delta t} = R(U^{n+1})\]

Implemented as a special case of the general unsteady formulation with \(\theta = 1\), \(\phi = 0.5\).

Input:

NUM_TIME_INT      = 3PB
NUM_DUAL_TIME     = T
NUM_RUN_MODE      = CONSTANT_DT
NUM_DT            = 1.0e-6
NUM_DUAL_TIME_MAX_IT = 20

Source files:

  • class_tpb_dt.F90, tpb_dt.F90 (FULL)

  • class_tpb_dt_diag.F90, tpb_dt_diag.F90 (DIAG)

Crank-Nicolson

Second-order implicit time-accurate method:

\[\frac{U^{n+1} - U^n}{\Delta t} = \frac{1}{2}\bigl[R(U^{n+1}) + R(U^n)\bigr]\]

Implemented with \(\theta = 0.5\), \(\phi = 0\).

Input:

NUM_TIME_INT      = CN
NUM_DUAL_TIME     = T
NUM_RUN_MODE      = CONSTANT_DT
NUM_DT            = 1.0e-6

Source files: class_cn_dt.F90, cn_dt.F90

Note

Only FULL LHS approximation is available for Crank-Nicolson.

Run Modes and CFL Control

Two run modes control how the time step is determined:

Keyword

Description

CONSTANT_CFL

Local time step computed per cell from CFL/VNN condition. Used for steady-state acceleration (time steps vary spatially).

CONSTANT_DT

Global uniform time step. Required for time-accurate and dual-time simulations.

CFL-related parameters:

NUM_CFL      = 1.0        # CFL number
NUM_VNN      = 0.5        # Von Neumann number (viscous stability)
NUM_RUN_MODE = CONSTANT_CFL

CFL ramping: The CFL number can be ramped during the simulation using a CFL table file specified in the boundary condition file. This allows gradual increase of CFL from a safe initial value to a large target value for steady-state convergence.

RHS time-step options:

Mode

Description

RHS_TIME_STEP_MULT_GLOBAL

RHS multiplied by global \(\Delta t\)

RHS_TIME_STEP_MULT_LOCAL

RHS multiplied by local \(\Delta t\) (per cell)

RHS_NO_TIME_STEP_MULT_VOL_DIV

No time-step multiplication (steady formulation)
